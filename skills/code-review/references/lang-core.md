# 跨语言通用代码级审计项

## L1 未使用的声明

- 未使用的 import / 局部变量 / 参数 / 字段
- 定义了但从未实例化或引用的类型（类/接口/type/struct/enum variant）
- 泛型参数定义了但函数体内未使用
- 仅在定义侧存在、无任何调用方的函数/方法
- 不可达分支（永真/永假条件）

## L1 未使用的依赖

- 包管理器（Cargo.toml / pubspec.yaml / pom.xml / package.json）中声明但代码中未引用的依赖
- dev 依赖声明了但测试/构建脚本中未使用

## L1 历史债务

- 超过 6 个月的 TODO / FIXME
- 标记 @deprecated 但仍在调用的代码
- 已弃用的 SDK / 框架 API 仍在使用

## L2 无用功路径

- 纯透传中间层：A→B→C，B 无任何加工
- 过度封装：仅一个实现的接口/抽象类且无扩展计划
- 重复逻辑：多处相同校验/计算——先判断重复是否源于同一职责（业务规则 / 架构关注点 / 通用能力），是则提取并按职责归位；形似但各自独立演化的不应合并
- 冗余防御：上游已保证的条件被下游重复检查
- 空 catch / 空回调 / 空实现（吞掉异常或事件）

## L2 冗余类型操作

- 无意义的类型转换链（已是目标类型仍做转换）
- 重复序列化/反序列化（同一数据反复编解码）

## L2 异步标记

- 标记 async 但内部无 await 的函数（应改为同步）

## L5 敏感信息

- 密钥/连接串/Token 硬编码在源码中（应走环境变量或密钥管理）
- 日志中打印密码、Token、身份证号等敏感信息
- 响应/输出中泄露不应返回的内部字段
- .git / .env 等敏感文件被部署到生产环境或提交到仓库

## L5 注入风险

- 字符串拼接构造 SQL / shell 命令 / 模板（注入风险）
- 用户输入未经校验和转义直接使用
- 文件操作未校验路径（路径穿越风险）

## L5 反序列化

- 反序列化未限制可实例化的类型（RCE 风险）
- 反序列化未设置 depth / size 上限（OOM 风险）

## L5 依赖安全

- 存在已知 CVE 高危版本的依赖
- 锁文件未提交到版本控制
- 关键依赖版本过旧（主版本落后 2+）
