# CLI 项目专项审计（命令行工具）

## 命令与子命令
- 子命令的 help text / description 缺失或与实际行为不符
- 命令别名（alias）定义了但与其他命令冲突

## 参数解析
- 定义了但从未被任何命令使用的全局 flag
- 参数定义了默认值但代码中总是显式传递（默认值无意义）
- 互斥参数未做互斥校验（如 --json 和 --table 同时传入）
- 必选参数标记为 optional 但代码中 unwrap（应标为 required）
- 环境变量回退定义了但代码中只读命令行参数

## 输入输出
- stdin/stdout/stderr 使用不规范（数据输出到 stderr、错误输出到 stdout）
- 输出格式（json/table/plain）定义了但某些命令只实现了部分格式
- 进度条/spinner 在非 TTY 环境未降级为简单日志
- 颜色输出未尊重 NO_COLOR / TERM 环境变量
- 交互式 prompt 在 pipe/非交互模式下阻塞

## 文件与系统操作
- 临时文件创建了但未在退出时清理（含 panic/中断场景）
- 文件路径未做平台兼容处理（硬编码 / 或 \）
- 配置文件路径查找逻辑中有永远不会命中的 fallback 路径
- 锁文件机制实现了但未处理 stale lock

## 退出与信号
- 退出码定义了多个值但实际只用了 0 和 1
- SIGINT/SIGTERM handler 注册了但清理逻辑不完整
- panic 未被捕获转为友好错误信息和非零退出码

## CLI 安全专项
- 凭证/Token 通过命令行参数传递（会出现在进程列表和 shell 历史中）
- 配置文件权限未检查（敏感配置被其他用户可读）
- 自动更新机制未验证下载包的签名/校验和
- 插件/扩展加载未做沙箱隔离或来源验证

## 分发与安装
- 安装脚本中残留的调试逻辑
- man page / shell completion 生成了但未包含在分发包中
- 多平台构建矩阵中有从未发布的 target
