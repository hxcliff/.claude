# 跨项目类型通用架构级审计项

## L1 入口与注册表

- 注册了但实现已删除或无任何调用方的入口点（API 端点 / 子命令 / 路由 / 事件监听）
- 入口参数（路由参数 / 命令 flag / 查询参数）声明了但 handler 中未读取

## L1 资源文件

- 未引用的 i18n key、翻译文件、静态资源（图片/字体/图标）
- 配置文件中定义但从未读取的字段、环境变量

## L2 空实现

- handler / controller / 子命令处理函数为空实现或返回 todo / placeholder
- 中间件/拦截器/Filter 注册了但实际是 pass-through

## L2 重复结构

- 多个 handler / 组件重复相同的校验/鉴权/格式化逻辑——先判断重复源于同一职责还是巧合相似，是则提取并按职责归入对应层级（中间件 / 业务 helper / utils）
- 结构体/组件 90%+ 代码重复，仅少量差异——先判断重复是否源于同一概念（业务实体 / 架构角色），是则合并或抽取公共部分并明确归属层级；各自独立演化的不应强行合并

## L2 已废弃功能

- 废弃的 API 版本 / 子命令 / 路由未从注册表移除
- 功能已下线但关联的配置、定时任务、消费者仍在运行

## L3 模块依赖健康度

- 循环依赖链（含通过延迟加载掩盖的隐式循环）
- 依赖方向倒置：底层工具模块反向引用业务模块
- 孤岛模块：无任何消费者的导出文件
- 公共模块膨胀：一个 utils/common 承载了不相关职责
- 跨层穿透：表现层直接访问数据层，跳过业务层

## L4 业务模型闭环检查

**流程完整性**
- 每条核心业务流是否有完整的正向+逆向路径
- 异常中断后是否有回滚/补偿机制
- 异步流程是否有超时兜底和重试上限
- 状态机是否存在死锁状态（进入后无法流转出去）

**数据一致性**
- 跨表/跨服务更新是否有事务或最终一致性保障
- 缓存与数据源之间是否有失效策略
- 并发场景下共享资源是否有正确的锁机制
- 删除操作是否级联清理了关联数据

**边界条件**
- 金额/数量是否处理了零值、负值、极大值、精度溢出
- 批量接口是否有上限防护
- 时区/时间逻辑是否统一标准
- 枚举/状态字段是否覆盖了所有可能值（含 default/未知态）

**第三方依赖闭环**
- 外部 API 调用是否有超时、熔断、降级
- 回调/webhook 是否有签名验证和幂等处理

## L5 认证与授权

- 是否存在未鉴权的接口/入口
- 权限校验是否仅在客户端而服务端缺失
- Token/Session 是否有过期和登出失效机制
- 多租户场景下是否有数据越权风险
- 密码是否使用安全哈希（bcrypt/scrypt/argon2）

## L5 接口防护

- 关键接口是否有频率限制（防暴力破解/DDoS）
- CORS / 同源策略是否正确配置
- 传输层是否强制加密（TLS）
- 文件上传是否校验了类型/大小/内容

## L6 全局健康度

- 构建脚本中注释掉或永不执行的步骤
- CI/CD 中跑了但结果被忽略的 job
- 容器镜像中安装了运行时不需要的包
- 监控/告警配置了但目标已下线
