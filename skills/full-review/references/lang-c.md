# C 语言专项审计

## L1 死代码

### 预处理器
- #define 宏定义了但未引用
- #ifdef / #if 分支条件永不为真（宏未定义或恒为零）
- 头文件声明了函数但无对应实现（链接时才暴露）

### 构建
- Makefile / CMakeLists.txt 中列出的源文件已删除
- 静态库编译了但未被链接

## L3 依赖健康

### 预处理器
- 头文件循环包含

## L5 安全

### 内存管理
- malloc / calloc / realloc 无配对 free（内存泄漏）
- 释放后仍有指针引用（悬垂指针）
- 重复 free（double free）
- 栈上 VLA 大小由用户输入决定（栈溢出风险）
- realloc 返回值未检查（原指针可能已失效）
- malloc 返回值未检查 NULL

### 指针与数组
- 数组越界访问（静态可推导的索引超界）
- 指针算术超出分配范围
- 函数返回局部变量地址
- void* 转换缺少大小校验
- 未初始化的指针解引用
- 函数指针类型不匹配

### 类型与表达式
- 隐式整型截断（将大类型赋值给小类型）
- 有符号 / 无符号混用比较
- 位运算对有符号整数（未定义行为）
- sizeof 对指针而非数组（得到指针大小而非数组大小）
- 枚举值隐式转换为整数后参与不安全运算
- 浮点数相等性比较（应使用 epsilon）

### C 安全专项
- 格式化字符串漏洞：printf / sprintf 的格式串来自用户输入
- 缓冲区溢出：strcpy / sprintf / gets / strcat 未限制长度
- 整数溢出未检查（尤其用于分配大小计算时）
- TOCTOU 竞态：access() 后 open()，stat() 后操作
- 未初始化变量读取
- signal handler 中调用非 async-signal-safe 函数
- setuid 程序未清理环境变量

## L6 全局健康

### 预处理器
- include guard 或 #pragma once 缺失（重复包含风险）

### 构建
- 编译 flag 禁用了关键警告（-Wno-error / -w）
- 目标文件依赖关系不完整（头文件修改不触发重编）
